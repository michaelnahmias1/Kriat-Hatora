# קריאת התורה — כתוביות מסונכרנות

הפקת קובץ כתוביות (SRT) עם טקסט מקראי **מנוקד ומוטעם**, מסונכרן להקלטת קריאה בטעמים,
לייבוא ל-CapCut. הערך של המערכת הוא הטקסט המדויק — לא התזמון (שכלי עריכה כבר עושים, אבל
תוך שיבוש העברית המקראית).

**התוכנית המלאה: [PLAN.md](PLAN.md)** ‏(v6 — הפייפליין ההיברידי).

## הממשק הקבוע — הכל מהטלפון, בלי Colab

**‏https://kriat-hatora.vercel.app**

1. **בחר קטע** — ספר, פרק, טווח פסוקים. הטקסט (גרסת MAM, מנוקד ומוטעם) נטען מ-Sefaria
   ונחתך למקטעים לפי טעמי המקרא.
2. **בחר את ההקלטה** מהטלפון — הקובץ נשאר במכשיר (מועלה רק אם בוחרים בתזמון הענן).
3. **תזמן**:
   - **🛰️ תזמון AI בענן** — המסלול ההיברידי המלא (תמלול ← עוגנים ← יישור ← VAD)
     רץ על GPU בענן ישירות מהאפליקציה, בלי Colab; הזמנים ודוח האיכות חוזרים
     ישר לשלב הבדיקה. (דורש פריסה חד-פעמית של ה-worker — ראה בהמשך.)
   - **⚡ תזמון אוטומטי במכשיר** — ניתוח ההקלטה בדפדפן (`autotime.js`), בלי העלאה.
   - תזמון ידני בהקשות וחלוקה לפי אורך נשארו כגיבוי.
4. **בדוק ותקן** — נגינה מכל נקודת כניסה, כוונון של ±0.2 שניות.
5. **ייצא SRT** (הורדה / שיתוף / העתקה) ← CapCut Web ← Cloud ← אפליקציית CapCut ← פרסום.

הממשק רץ על Vercel (פרויקט `kriat-hatora`): דף סטטי + פונקציית Python אחת
(`api/segments.py`) שמושכת את הטקסט מ-Sefaria וחותכת אותו עם `src/chunker`.
**אין צורך באחסון ענן** (Supabase וכד') — עיבוד הענן הוא job חולף שלא שומר כלום.

## מסלול היישור המדויק (AI) — הפייפליין ההיברידי

המסלול שנותן את הסנכרון הטוב ביותר, בשלוש שכבות שכל אחת מפצה על חולשת קודמתה:
תמלול עברי (ivrit.ai) שמוצלב עם טקסט Sefaria ומייצר "עוגנים", יישור כפוי (MMS)
על חלונות קצרים בין העוגנים, והצמדת גבולות לנשימות (VAD). לצד ה-SRT נוצר דוח
איכות שמפרט אילו מקטעים בודדים כדאי לגרור ידנית.

### בתוך האפליקציה (Modal) — המסלול הראשי

הצינור רץ כ-worker ב-[Modal](https://modal.com) ‏(GPU T4 לפי-שימוש; התוכנית
החינמית כוללת קרדיט חודשי שמכסה שימוש אישי בשפע). האפליקציה שולחת את ההקלטה
ואת הפרמטרים ל-endpoint, שואלת לשלום העבודה כל כמה שניות, ובסיום מזרימה את
הזמנים ודוח האיכות ישר לשלב הבדיקה. **הקוד ב-`src/aligner` זהה לזה של Colab —
אותה משיכת טקסט מ-Sefaria ואותה שיטת עיבוד בדיוק.**

פריסה חד-פעמית (ממחשב, עם חשבון Modal — הרשמה עם GitHub):

```bash
pip install modal
modal setup                                              # התחברות (פעם אחת)
modal secret create kriat-hatora KRIAT_HATORA_TOKEN=<מחרוזת-אקראית-ארוכה>
modal deploy modal_worker/app.py                         # מדפיס את כתובת ה-web
```

ואז ממלאים ב-`index.html` את שני הקבועים שבראש הסקריפט — `AI_ENDPOINT` (הכתובת
שהודפסה, של הפונקציה `web`) ו-`AI_TOKEN` (אותה מחרוזת מה-secret) — ופורסים ל-Vercel
כרגיל. כל עוד הקבועים ריקים הכפתור פשוט מוסתר והאפליקציה עובדת כמו קודם.

הערה: ה-token חשוף בצד הלקוח — זו הגנת-עלות בסיסית מפני שימוש אקראי בכתובת,
לא סוד אמיתי. מתאים לכלי אישי; אם הכלי ייפתח לציבור, נוסיף שער אמיתי.

**חשוב — שתי פריסות נפרדות:** דחיפה ל-GitHub מעדכנת אוטומטית רק את האתר
(Vercel). ה-worker בענן **לא** מתעדכן מדחיפה — בלי פריסה מחדש הוא ממשיך
להריץ את הקוד הישן, גם כששינוי כבר נראה באתר (כך שגיאת ה-torchcodec חזרה
אחרי שכבר תוקנה בקוד). שתי דרכים לעדכן אותו:

- **אוטומטית (מומלץ):** ה-workflow‏ `.github/workflows/deploy-modal.yml` פורס
  ל-Modal בכל שינוי של `src/` או `modal_worker/` שמגיע ל-main. הפעלה
  חד-פעמית: יוצרים token ב-Modal‏ (Settings ← API Tokens, או `modal token new`)
  ומוסיפים בריפו ב-GitHub‏ (Settings ← Secrets and variables ← Actions) את
  שני ה-secrets‏ `MODAL_TOKEN_ID` ו-`MODAL_TOKEN_SECRET`.
- **ידנית:** מריצים שוב `modal deploy modal_worker/app.py` מהמחשב אחרי כל
  שינוי בצינור.

### ב-Colab (גיבוי)

אותו מעבד בדיוק, למקרה שהענן לא זמין:

1. **פתח את המעבד ב-Colab** (יש קישור גם בתחתית האפליקציה):

   [![Open In Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/michaelnahmias1/Kriat-Hatora/blob/main/notebooks/worker.ipynb)

   בפעם הראשונה: ‏Runtime ← Change runtime type ← **T4 GPU**.
2. **מלא את תא הפרמטרים** (ספר, פרק, טווח פסוקים) ← Runtime ← **Run all** ←
   כשתא 2 מבקש, **בחר את קובץ ההקלטה ישירות מהטלפון** (לא צריך Google Drive;
   מי שבכל זאת עובד מ-Drive — ממלא נתיב בתא 1 כמו קודם).
3. ה-SRT ודוח האיכות מוצעים להורדה ישירה לטלפון, וייבוא ה-SRT בשלב 3 באפליקציה
   מכניס את הזמנים לשלב הבדיקה.
